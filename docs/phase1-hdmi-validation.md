# 阶段一 HDMI 验证清单

## 1. 验证目标

阶段一使用 Radxa CM3 IO Board + 4.3 寸 800x480 HDMI 屏验证 RK3566 掌机方案。此阶段不处理 DSI 点屏、自研底板和最终外壳，只验证系统、显示、输入、音频、散热、功耗和模拟器体验。

第一优先目标是 GBA。GBA 达标后，再扩展到 PS1、SFC/MD/FC/街机、PSP、N64 和 Dreamcast 的边界测试。

阶段一需要回答的问题：

- RK3566 + Linux 是否能稳定提供 GBA 掌机体验。
- 4.3 寸 800x480 横屏是否适合当前掌机方向。
- USB 手柄、临时按键、音频、散热和功耗是否满足进入成品规格定义的基础。
- PSP、N64、Dreamcast 是否值得纳入成品阶段目标。

## 3. 屏幕选择检查

阶段一优先使用 HDMI 屏，屏幕需满足：

- HDMI 视频输入。
- 800x480 或可稳定输出 800x480。
- USB 供电。
- 如果带触摸，触摸应为 USB HID。
- 不依赖 Raspberry Pi GPIO、SPI 或专用 overlay。

不建议阶段一使用：

- SPI 显示屏。
- 依赖 GPIO/SPI 的触摸屏。
- 需要定制驱动板或特殊 EDID 的屏。
- 高价 1080p AMOLED 屏。

## 8. GBA 验证

GBA 是阶段一最优先目标。建议先验证 mGBA 独立版，再验证 RetroArch mGBA core。

### 8.1 显示策略

GBA 原生分辨率为 240x160。800x480 横屏下优先使用 3x 整数缩放：

- 画面区域：720x480。
- 左右黑边：各 40 px。
- 垂直方向：刚好占满 480 px。

## 13. 阶段一通过标准

阶段一通过需要满足：

- GBA 可稳定 60 FPS。
- GBA 显示策略明确，优先 3x 整数缩放。
- 至少一种输入方案完整可用。
- 至少一种音频方案稳定可用。
- 30 分钟 GBA 稳定性测试通过。
- 温度和功耗没有明显阻碍。
- 屏幕尺寸和分辨率方向有明确结论。

阶段一结束时应输出：

- 推荐系统镜像。
- 推荐 GBA 模拟器和配置。
- 推荐输入映射。
- 推荐音频路径。
- 屏幕尺寸结论。
- 散热和功耗结论。
- 是否进入成品底板设计。
